📌 Cheat Sheet Symfony & Git

Commandes terminal pour développer, corriger et tester une application Symfony

🔹 Git : Gestion du code et des commits

1. Statut et historique

Commande Description
git status Affiche l’état des fichiers (modifiés, non suivis, etc.).
git log --oneline Affiche l’historique des commits (version compacte).
git log -p Affiche l’historique avec les différences (diffs).
git show <commit-hash> Affiche les détails d’un commit spécifique.

2. Manipulation des commits

Commande Description
git add <fichier> Ajoute un fichier à l’index (staging).
git add . Ajoute tous les fichiers modifiés.
git commit -m "message" Crée un commit avec un message.
git commit --amend Modifie le dernier commit (message ou contenu).
git reset --soft HEAD~1 Annule le dernier commit mais conserve les modifications dans le working directory.
git reset --hard HEAD ⚠️ Supprime toutes les modifications non commitées et revient au dernier commit.
git reset --hard HEAD~1 ⚠️ Supprime le dernier commit ET ses modifications.
git clean -fd Supprime tous les fichiers non suivis (y compris les dossiers).

3. Branches

Commande Description
git branch Liste les branches locales.
git branch <nom-branche> Crée une nouvelle branche.
git checkout <nom-branche> Change de branche.
git checkout -b <nom-branche> Crée et bascule sur une nouvelle branche.
git merge <nom-branche> Fusionne une branche dans la branche courante.
git branch -d <nom-branche> Supprime une branche locale.
git push origin --delete <nom-branche> Supprime une branche distante.
git pull origin <nom-branche> Récupère les changements d’une branche distante.

4. Collaboration (Remote)

Commande Description
git clone <url> Clone un dépôt distant.
git remote -v Affiche les dépôts distants configurés.
git push origin <nom-branche> Pousse une branche vers le dépôt distant.
git pull Récupère et fusionne les changements distants.
git fetch Récupère les changements distants sans fusionner.
git rebase <nom-branche> Réapplique les commits de la branche courante sur une autre branche.

5. Résolution de conflits

Commande Description
git diff Affiche les différences entre le working directory et l’index.
git diff <commit1> <commit2> Compare deux commits.
git stash Sauvegarde temporairement les modifications non commitées.
git stash pop Récupère les modifications sauvegardées avec stash.
git stash list Liste toutes les sauvegardes stash.

🔹 Symfony : Développement et Tests

1. Installation et Mise à jour

Commande Description
composer install Installe les dépendances du projet (via composer.lock).
composer update Met à jour les dépendances (modifie composer.lock).
composer require <package> Ajoute une dépendance au projet.
composer remove <package> Supprime une dépendance.
symfony check:requirements Vérifie que l’environnement répond aux exigences de Symfony.

2. Serveur et Cache

Commande Description
symfony serve Lance le serveur de développement local (port 8000 par défaut).
symfony serve -d Lance le serveur en arrière-plan (mode démon).
symfony serve:stop Arrête le serveur.
php bin/console cache:clear ⚠️ Vide le cache (à utiliser après des modifications de configuration).
php bin/console cache:warmup Préchauffe le cache.

3. Génération de Code

Commande Description
php bin/console make:controller <NomController> Crée un nouveau contrôleur.
php bin/console make:entity <NomEntity> Crée une nouvelle entité Doctrine.
php bin/console make:form <NomForm> Crée un nouveau formulaire.
php bin/console make:crud <NomEntity> Génère un CRUD complet pour une entité.
php bin/console make:user Crée une entité User avec authentification.
php bin/console make:auth Configure l’authentification (login, registration).
php bin/console make:command <NomCommand> Crée une commande CLI personnalisée.

4. Base de Données (Doctrine)

Commande Description
php bin/console doctrine:database:create Crée la base de données.
php bin/console doctrine:schema:update --force Met à jour le schéma de la base de données.
php bin/console doctrine:migrations:diff Génère une migration à partir des entités.
php bin/console doctrine:migrations:migrate Exécute les migrations en attente.
php bin/console doctrine:migrations:execute <nom-migration> --up Exécute une migration spécifique.
php bin/console doctrine:fixtures:load Charge les fixtures (données de test).

5. Tests

Commande Description
php bin/phpunit Lance tous les tests unitaires et fonctionnels.
php bin/phpunit --filter <NomTest> Lance un test spécifique.
php bin/phpunit --group <groupe> Lance les tests d’un groupe spécifique.
php bin/console debug:test Affiche la liste des tests disponibles.

6. Debug et Inspection

Commande Description
php bin/console debug:router Affiche toutes les routes de l’application.
php bin/console debug:container Liste tous les services du conteneur.
php bin/console debug:autowiring Affiche les services disponibles pour l’autowiring.
php bin/console debug:config <service> Affiche la configuration d’un service.
php bin/console profiler:dump Affiche les dernières requêtes profilées.

7. Assets et Frontend

Commande Description
npm install Installe les dépendances Node.js (si tu utilises Webpack Encore).
npm run dev Compile les assets en mode développement.
npm run build Compile les assets en mode production.
npm run watch Compile les assets automatiquement à chaque modification.

8. Sécurité

Commande Description
php bin/console security:check Vérifie les vulnérabilités des dépendances.
php bin/console debug:firewall Affiche la configuration des pare-feux.
php bin/console debug:access <route> Vérifie les permissions d’accès à une route.

🔹 Workflows Courants

1. Corriger une erreur non commité

git reset --hard HEAD # Supprime toutes les modifications non commitées
git clean -fd # Supprime les fichiers non suivis (optionnel)

2. Corriger un bug dans une nouvelle branche

git checkout -b fix/nom-du-bug
# Faire les corrections...
git add .
git commit -m "Fix: description du bug"
git push origin fix/nom-du-bug

3. Tester une fonctionnalité

php bin/phpunit --filter NomDuTest
php bin/console cache:clear
symfony serve

4. Mettre à jour une dépendance

composer update <package> --with-dependencies
php bin/console cache:clear

5. Déployer en production

composer install --no-dev --optimize-autoloader
php bin/console doctrine:migrations:migrate
php bin/console cache:clear --env=prod --no-debug
npm run build

💡 Astuces

Aliases Git : Ajoute ces alias dans ton ~/.bashrc ou ~/.zshrc pour gagner du temps :

alias gs="git status"
alias ga="git add ."
alias gc="git commit -m"
alias gp="git push"
alias gl="git log --oneline"
alias gb="git branch"
alias gco="git checkout"

Symfony CLI : Installe Symfony CLI pour une meilleure expérience locale.

Debug Bar : Active la barre de debug en développement avec :

# config/packages/dev/web_profiler.yaml
web_profiler:
toolbar: true
intercept_redirects: false
